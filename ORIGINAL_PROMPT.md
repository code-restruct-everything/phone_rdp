====================
一、最终目标
====================
开发一个 Android 手机端 RDP 客户端（MVP），能连接远程 Windows 电脑并进行基本操作。

核心验收：
1. 可输入并保存连接配置（host/ip、port、username、password、domain 可选）。
2. 可建立/断开 RDP 会话。
3. 连接后可显示远程桌面画面。
4. 支持基础输入映射：
   - 单击 = 左键
   - 长按 = 右键
   - 双指拖动 = 滚轮
   - 双指缩放 = 本地视图缩放（不改远端分辨率）
5. 支持软键盘输入文本。
6. 连接失败/断连有可读错误提示，支持一键重连。
7. 最近连接至少保存 5 条，密码必须加密存储。

====================
二、技术约束（强制）
====================
1. 平台：只做 Android，不做 iOS。
2. 业务与 UI：Kotlin + Jetpack Compose。
3. RDP 内核：必须使用 FreeRDP（JNI/NDK 接入），禁止从零实现协议栈。
4. 代码分层必须清晰：
   - ui
   - domain
   - data
   - native_bridge
5. 不引入重型无关依赖。
6. 关键桥接点要有简短注释。
7. 不要求 Java 业务代码；允许 Kotlin + C/C++。
8. 构建基线：JDK 17，Gradle Kotlin DSL。

====================
三、安全与稳定性要求
====================
1. 密码不得明文落盘。
2. 使用 Android Keystore + EncryptedSharedPreferences（或等效方案）保存敏感数据。
3. 日志默认脱敏，不打印明文密码和完整凭据。
4. 支持 TLS/NLA 场景，证书/认证异常需要分类提示。
5. JNI 层需要基础错误边界处理，避免 native 崩溃直接导致无提示闪退。

====================
